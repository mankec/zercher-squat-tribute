How computed CSS styles of a page element relate to the original CSS in a project.

Relevant files:

- `src/routes/+page.svelte`
- `src/lib/components/Navbar.svelte`
- `src/routes/about/+page.svelte`

First thing I noticed is that my CSS from a `<style>` has been renamed into gibberish set of letters. From `div` to `s-y_bCXRrkrYfP` and it's been added to the end of every `div`'s `class` tag. First I thought that this random set of letters will be generated for each element separately but it turns out that this represent all of the CSS i added in `<style>`. Well turns out SvelteKit automatically adds it to every element in a file if it detects `<style>`. For example `Navbar` component will not have it because there's no `<style>` tag in its file. What Svelte does is it takes my `a` override and attaches coded name to it so it looks like this for example `a.s-y_bCXRrkrYfP` that way only `a`'s in this file are overriden. Neat! To confirm this I added `<style>` to about page and coded style name was different than the one in the root page file
